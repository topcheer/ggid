# Docker Compose Deployment Guide

This guide covers deploying the full GGID IAM suite using Docker Compose — from initial setup to E2E testing and troubleshooting.

## Prerequisites

- Docker Engine 24.0+
- Docker Compose v2.20+
- 4 GB RAM minimum (8 GB recommended)
- Ports: 8080, 8081, 8082, 9001, 9005, 8070, 8071, 8072, 3000

## Architecture

GGID runs 12 containers in Docker Compose:

| Container          | Image               | Host Port | Purpose                      |
|--------------------|---------------------|-----------|------------------------------|
| ggid-postgres      | postgres:16-alpine  | 5432      | PostgreSQL 16 database       |
| ggid-redis         | redis:7-alpine      | 6379      | Redis cache & sessions       |
| ggid-nats          | nats:2-alpine       | 4222/8222 | NATS JetStream messaging     |
| ggid-ldap          | osixia/openldap:1.5 | 389       | OpenLDAP test directory      |
| ggid-keygen        | alpine:3.20         | —         | One-shot: generate TLS keys  |
| ggid-migrate       | golang:1.25-alpine  | —         | One-shot: run DB migrations  |
| ggid-identity      | deploy-identity     | 8081      | Identity service             |
| ggid-auth          | deploy-auth         | 9001      | Authentication service       |
| ggid-oauth         | deploy-oauth        | 9005      | OAuth/OIDC service           |
| ggid-gateway       | deploy-gateway      | 8080      | API gateway (entry point)    |
| ggid-policy        | deploy-policy       | 8070/9070 | Policy/authorization service |
| ggid-org           | deploy-org          | 8071/9071 | Organization service         |
| ggid-audit         | deploy-audit        | 8072/9072 | Audit logging service        |

## Quick Start

```bash
cd deploy

# 1. Build all service images
docker compose build

# 2. Start the full stack
docker compose up -d

# 3. Wait for healthchecks to pass (~30s)
sleep 30

# 4. Run the E2E test suite
bash e2e-docker-test.sh
```

## Environment Variables

### Database

```yaml
DB_HOST: postgres
DB_PORT: 5432
DB_USER: ggid
DB_PASSWORD: ggid
DB_NAME: ggid
DB_SSLMODE: disable
```

> **Important**: Policy, Org, and Audit services use individual `DB_HOST`/`DB_PORT`/`DB_USER`/`DB_PASSWORD` env vars — NOT a single `DATABASE_URL`. This is a common misconfiguration.

### NATS

```yaml
NATS_URL: nats://nats:4222
```

NATS must be started with the `-m 8222` flag for the monitoring port used by healthchecks.

### Redis

```yaml
REDIS_ADDR: redis:6379
REDIS_PASSWORD: ""
```

### JWT

```yaml
JWT_SECRET: <auto-generated by ggid-keygen>
JWT_ISSUER: ggid
```

The `ggid-keygen` one-shot container generates RSA key pairs on first run and stores them in a named Docker volume. Do NOT leave `JWT_SECRET` empty in production.

### LDAP (optional)

```yaml
LDAP_URL: ldap://ldap:389
LDAP_BIND_DN: cn=admin,dc=ggid,dc=local
LDAP_BIND_PASSWORD: admin
LDAP_BASE_DN: dc=ggid,dc=local
LDAP_USER_FILTER: (uid=%s)
LDAP_START_TLS: "false"
LDAP_AUTO_PROVISION: "true"
```

When `LDAP_URL` is empty, the auth service runs in LocalProvider-only mode.

## Default Tenant

All API requests through the gateway must include the `X-Tenant-ID` header:

```
X-Tenant-ID: 00000000-0000-0000-0000-000000000001
```

## E2E Test Suite

The `e2e-docker-test.sh` script runs 11 tests:

| # | Test                    | Expected | Description                              |
|---|-------------------------|----------|------------------------------------------|
| 1 | Gateway healthz         | 200      | Health endpoint responds                 |
| 2 | Register user           | 201      | User registration with username field    |
| 3 | Login + JWT             | 200      | Login returns 600+ char JWT              |
| 4 | 401 without JWT         | 401      | Unauthenticated request rejected         |
| 5 | List users              | 200      | Identity users endpoint works            |
| 6 | Create role             | 201      | Policy role creation (needs `key` field) |
| 7 | List roles              | 200      | Policy roles listing                     |
| 8 | Create org              | 201      | Organization creation                    |
| 9 | Audit query             | 200      | Audit event query                        |
| 10| Wrong password          | 401      | Incorrect credentials rejected           |
| 11| Duplicate register      | 409      | Duplicate username returns Conflict      |

### Running E2E Tests

```bash
bash deploy/e2e-docker-test.sh
```

Expected output: `11/11 ALL PASS`

## Health Checks

Each service exposes a `/healthz` endpoint. Docker Compose healthchecks poll these endpoints:

```yaml
healthcheck:
  test: ["CMD", "wget", "--spider", "-q", "http://localhost:PORT/healthz"]
  interval: 10s
  timeout: 5s
  retries: 3
  start_period: 15s
```

Check status:

```bash
docker compose ps
# Look for "healthy" status on all containers
```

## Common Operations

### View Logs

```bash
# All services
docker compose logs -f

# Single service
docker compose logs -f ggid-gateway
docker compose logs -f ggid-auth
```

### Restart a Service

```bash
docker compose restart ggid-auth
```

> Auth rate-limits after ~5 failed login attempts. Restart the container to clear the rate limit counter.

### Run Migrations Manually

The `ggid-migrate` container is idempotent — it skips if tables already exist. To force re-run:

```bash
docker compose run --rm ggid-migrate
```

### Rebuild After Code Changes

```bash
docker compose build ggid-auth
docker compose up -d ggid-auth
```

## Troubleshooting

### Container Won't Start

1. Check logs: `docker compose logs ggid-<service>`
2. Verify dependencies are healthy: `docker compose ps`
3. Common causes:
   - **PostgreSQL not ready**: Migration container fails. Re-run `docker compose run --rm ggid-migrate` after Postgres is healthy.
   - **NATS monitoring port**: NATS must be started with `-m 8222` flag.
   - **Port conflict**: Ensure ports 8080, 5432, 6379, 4222 are not in use.

### 502/504 from Gateway

Gateway can't reach upstream services. Check:

```bash
# Verify services are running
docker compose ps ggid-identity ggid-auth ggid-policy

# Check if services are listening on correct ports
docker exec ggid-gateway wget -qO- http://identity:8080/healthz
```

### Database Connection Errors

Policy, Org, Audit services use `DB_HOST`/`DB_PORT` env vars (NOT `DATABASE_URL`). Verify:

```bash
docker exec ggid-policy env | grep DB_
```

Expected: `DB_HOST=postgres`, `DB_PORT=5432`, `DB_USER=ggid`, etc.

### Rate Limiting (429)

Auth service rate-limits after ~5 failed login attempts:

```bash
docker compose restart ggid-auth
```

### E2E Test Failures

| Test          | Failure Cause                    | Fix                                         |
|---------------|----------------------------------|---------------------------------------------|
| Create role   | Missing `key` field              | Role needs unique `key` (UNIQUE constraint) |
| Register      | Missing `username` field         | Register reads `username`, not `email`      |
| Audit query   | 404 route mismatch               | Gateway routes `/api/v1/audit` → audit alias|
| Login         | 429 rate limited                 | Restart auth container                      |

### Certificate / TLS Issues

The `ggid-keygen` container generates self-signed certs on first run. If certs are corrupted:

```bash
docker compose down -v  # removes volumes (WARNING: destroys data)
docker compose up -d
```

## Production Considerations

- **Change all default passwords** (DB, Redis, LDAP, JWT secret)
- **Use external PostgreSQL** with automated backups
- **Enable TLS** for all inter-service communication (gRPC TLS is supported)
- **Set up log aggregation** (ELK, Datadog, or CloudWatch)
- **Configure SIEM forwarding** via the audit service
- **Enable RLS policies** for multi-tenant isolation
- **Set resource limits** in docker-compose.yaml for each container
