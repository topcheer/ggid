# GGID 产品设计方案

## 核心问题
825 个 Console 页面（500+ settings 子页面），功能堆砌，没有产品逻辑。

## 产品定位
GGID = 企业级 IAM/IIGA 平台，目标中型企业（100-10000人）。

## 四层角色体系

| 角色 | Scope | 职责 |
|------|-------|------|
| 平台超管 | platform:admin | 管理实例、租户 |
| 租户管理员 | tenant:admin | 管理租户用户/角色/安全 |
| 租户审计员 | tenant:auditor | 只读审计 |
| 普通用户 | user:self | 管理个人信息 |

## 初始化流程
1. Phase 1: 实例初始化 → 创建超管 + 默认租户
2. Phase 2: 超管创建租户 → 指定租户管理员
3. Phase 3: 租户管理员管理用户

## Console 精简目标
825 页面 → ~40 核心页面
