# Test Automation Guide

## Overview

GGID's automated test framework combines smoke tests, E2E business flows, and performance baselines into a unified suite. All scripts run on every PR via CI.

## Test Suite Architecture

```
CI Pipeline
├── Build & Test (make test — 4461 unit/integration tests)
├── Security Scan (govulncheck + gosec)
├── Lint (golangci-lint v1.64.8)
├── UI Smoke Check (page count + script validation)
└── E2E Script Check (syntax validation)
```

## Scripts

### 1. UI Smoke Test (`scripts/ui-smoke-test.sh`)

Verifies all 828 console pages return HTTP 200.

```bash
# Run against any deployment
bash scripts/ui-smoke-test.sh https://ggid-console.iot2.win

# Output: PASS/FAIL/BROKEN counts + markdown report
# Report: docs/test/ui-smoke-report.md
```

**Current baseline:** 822/828 PASS (99.3%)

### 2. E2E Business Flows (`scripts/e2e-business-flows.sh`)

Tests 11 core API flows with timing:

```bash

# Flows tested:
# 1. Login → token
# 2. Create User → 201

# 11. List Sessions → 200

# Output: PASS/FAIL + latency per step + markdown report
# Report: docs/test/e2e-business-flows-report.md
```

**Current baseline:** 11/11 PASS (100%)

## CI Integration

| Job | Trigger | What It Does |
|-----|---------|--------------|
| `lint` | Every push/PR | golangci-lint v1.64.8 |
| `mod-tidy` | Every push/PR | go mod tidy check |
| `build-test` | Every push/PR | go build + make test |
| `security-scan` | Every push/PR | govulncheck + gosec |
| `ui-smoke-check` | Every push/PR | Page count ≥700 + script syntax |
| `e2e-syntax-check` | Every push/PR | E2E script syntax validation |

## Pre-Commit Checklist

Before every commit:

```bash
go build ./...
make test                    # 0 FAIL
golangci-lint run --timeout=5m  # CI v1.64.8
```

## Running Against Remote

```bash
# Full verification suite
export GGID=https://ggid.iot2.win
bash scripts/ui-smoke-test.sh https://ggid-console.iot2.win
bash scripts/e2e-business-flows.sh $GGID
```

## Adding New Tests

1. **New page**: No action needed — smoke test auto-discovers `page.tsx` files
2. **New API endpoint**: Add a step to `e2e-business-flows.sh`
3. **New unit test**: Add `*_test.go` in the appropriate package
4. **New security test**: Add to `api_security_test.go` or `e2e_integration_test.go`
