# Round 62 E2E Findings — OAuth Service + JWT Scope

## Date: 2026-07-15

## P0-1: OAuth Service Unavailable

**Symptom:** POST /api/v1/oauth/clients returns "backend service unavailable"
GET /api/v1/oauth/clients works (returns empty list)

**Likely cause:** OAuth service starts but crashes or doesn't register POST routes properly.
Need to check oauth-server.err log for startup errors.

## P0-2: Admin JWT Missing Scope

**Symptom:** GET /api/v1/admin/feature-flags returns 403 "admin scope required"

**Root cause analysis:**
The JWT token generated by auth service doesn't include admin role/scope in claims.
The bootstrap creates admin user and assigns admin role, but the Login flow doesn't
read user roles when generating JWT.

**Fix needed:**
1. Auth service Login should query user roles from identity service
2. JWT claims should include `roles: ["admin"]` or `scope: "admin"`
3. Admin middleware should check for admin role in JWT
