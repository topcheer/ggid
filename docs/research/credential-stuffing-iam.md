## 1. Credential Stuffing Attack Mechanics

### 1.1 Definition: Stuffing vs Brute Force

| Aspect | Brute Force | Credential Stuffing |
|---|---|---|
| **Password source** | Generated by trying all combinations | Real passwords from data breaches |
| **Success rate** | ~0.001% per attempt (if bcrypt/scrypt) | 0.1%–2% per attempt |
| **Volume** | Billions of guesses (offline) | Thousands of login attempts (online) |
| **Target** | One account, many passwords | Many accounts, known passwords |
| **Defense** | Rate limiting, strong hashing | Breach detection, MFA, device fingerprinting |

Credential stuffing is devastating because it uses passwords users actually chose. When a
breach corpus contains 10 billion `email:password` pairs (Collection #1-5, 2019), an attacker
can spray these against any login endpoint. The 1-2% success rate means 1-2 accounts out of
every 100 attempted will fall — and a large tenant with 10,000 users loses 100-200 accounts.

### 1.2 Attack Tools

**Selenium / Puppeteer / Playwright**: Headless browsers that execute JavaScript, solve
basic CAPTCHAs, and appear indistinguishable from real browsers at the HTTP level. These
bypass naive bot detection that only checks for missing JavaScript execution.

**Credential Stuffing APIs**: Commercial services like `SneakerBot`, `AccountGen`, and
underground forums offer pre-configured stuffing tools with proxy rotation, CAPTCHA solving,
and breach-corpus databases. These "as-a-service" offerings lower the barrier to entry.

**Open-source tools**: `Sniper`, `STORM-BREAKER`, and custom Python scripts using
`requests` + `concurrent.futures` for parallel stuffing across distributed proxies.

### 1.3 Distributed Attacks Across IPs

Modern stuffing campaigns distribute requests across thousands of residential proxy IPs:

- **Residential proxy services** (Bright Data, Oxylabs, Smartproxy) — 70M+ IPs that appear
  as legitimate home connections, evading datacenter IP blocklists.
- **Botnet proxy networks** — compromised IoT devices and home routers route stuffing
  traffic, making each request appear from a different residential IP.
- **Rate limit evasion** — by rotating IPs, each IP sends only 1-2 requests, staying under
  per-IP thresholds (e.g., GGID's 5/min default).

### 1.4 Low-and-Slow Attacks

Instead of high-volume bursts, sophisticated attackers send 1 request per IP every few
minutes across a pool of 10,000+ proxies. This means:

- Per-IP rate limits never trigger (1 request per IP per hour)
- Per-account rate limits rarely trigger (1 attempt per account per hour)
- The attack runs for days/weeks, accumulating thousands of attempts
- Detection requires cross-IP pattern analysis, not per-IP thresholds

### 1.5 IP Rotation Services

| Service Type | Pool Size | Detection Difficulty |
|---|---|---|
| Datacenter proxies | 10K-100K | Easy (known ASN ranges) |
| Residential proxies | 1M-70M | Hard (legitimate ISP IPs) |
| Mobile proxies | 100K-5M | Very hard (legitimate carrier IPs) |
| Botnet residential | Unknown | Extremely hard (real infected devices) |

**Defense implication**: IP-based rate limiting alone is insufficient. Defense must combine
multiple signals: breach corpus detection, device fingerprinting, behavioral analysis, and
CAPTCHA escalation.

## 2. Breach Corpus Detection at Login

### 2.1 Why Check at Login, Not Just Registration

GGID currently checks passwords against the HIBP API during **registration only**
(`services/auth/internal/service/password_breach.go`). This leaves existing users vulnerable:

- A user registered before the check was added
- A user's password appears in a new breach after registration
- The user never changed their password

**Login-time breach detection** catches these cases. When a user successfully authenticates,
check whether their password is in a known breach. If so, force a password change before
granting a full session.

### 2.2 k-Anonymity HIBP API Integration at Login

The k-anonymity model sends only the first 5 characters of the SHA-1 hash to the HIBP API.
The API returns all breach suffixes matching that prefix (typically 500-800 entries). The
client matches locally. The HIBP server never learns enough to identify the password.

**Performance concern**: Calling an external API on every login adds latency. Mitigations:

1. **Cache results**: Cache the entire prefix response (5-char prefix → list of suffixes)
   in Redis with a 24-hour TTL. Only ~1M possible prefixes, so cache hit rate is high.
2. **Async check**: Check asynchronously after login; flag the account if breached; force
   reset on next login.
3. **Batch pre-load**: For high-traffic systems, pre-cache all 1M prefixes overnight.

### 2.3 Go Code: Login-Time Breach Check with Redis Caching

```go
package service

import (
	"context"
	"crypto/sha1"
	"encoding/hex"
	"fmt"
	"io"
	"net/http"
	"strings"
	"time"

	"github.com/redis/go-redis/v9"
)

const (
	breachCacheTTL    = 24 * time.Hour
	breachCachePrefix = "ggid:hibp:range:"
)

// BreachChecker performs k-anonymity HIBP breach lookups with Redis caching.
type BreachChecker struct {
	rdb    *redis.Client
	client *http.Client
}

func NewBreachChecker(rdb *redis.Client) *BreachChecker {
	return &BreachChecker{
		rdb: rdb,
		client: &http.Client{
			Timeout: 3 * time.Second, // fail fast; don't block login
		},
	}
}

// IsPasswordBreached returns true if the password appears in known breaches.
// Uses k-anonymity: only 5 chars of SHA-1 are sent to the API.
// Results are cached per-prefix in Redis for 24 hours.
func (bc *BreachChecker) IsPasswordBreached(ctx context.Context, password string) (bool, int, error) {
	// Compute SHA-1 hash.
	h := sha1.Sum([]byte(password))
	hash := strings.ToUpper(hex.EncodeToString(h[:]))
	prefix := hash[:5]
	suffix := hash[5:]

	// Try cache first.
	cacheKey := breachCachePrefix + prefix
	suffixes, err := bc.rdb.SMembers(ctx, cacheKey).Result()
	if err != nil || len(suffixes) == 0 {
		// Cache miss — fetch from HIBP API.
		suffixes, err = bc.fetchRange(ctx, prefix)
		if err != nil {
			// Fail open: don't block login if HIBP is unreachable.
			return false, 0, nil
		}
		// Cache the results.
		if len(suffixes) > 0 {
			pipe := bc.rdb.Pipeline()
			for _, s := range suffixes {
				pipe.SAdd(ctx, cacheKey, s)
			}
			pipe.Expire(ctx, cacheKey, breachCacheTTL)
			_, _ = pipe.Exec(ctx)
		}
	}

	// Check if our suffix is in the breached set.
	for _, s := range suffixes {
		// Cached entries may store "SUFFIX:COUNT" or just "SUFFIX".
		if idx := strings.Index(s, ":"); idx > 0 {
			if s[:idx] == suffix {
				var count int
				fmt.Sscanf(s[idx+1:], "%d", &count)
				return true, count, nil
			}
		} else if s == suffix {
			return true, 1, nil
		}
	}
	return false, 0, nil
}

// fetchRange retrieves the breach suffix list for a 5-char prefix from HIBP.
func (bc *BreachChecker) fetchRange(ctx context.Context, prefix string) ([]string, error) {
	url := "https://api.pwnedpasswords.com/range/" + prefix
	req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
	if err != nil {
		return nil, err
	}
	req.Header.Set("User-Agent", "GGID-IAM")

	resp, err := bc.client.Do(req)
	if err != nil {
		return nil, err
	}
	defer resp.Body.Close()

	if resp.StatusCode != http.StatusOK {
		return nil, fmt.Errorf("HIBP API returned %d", resp.StatusCode)
	}

	body, err := io.ReadAll(resp.Body)
	if err != nil {
		return nil, err
	}

	var result []string
	for _, line := range strings.Split(string(body), "\n") {
		line = strings.TrimSpace(line)
		if line != "" {
			result = append(result, line) // "SUFFIX:COUNT"
		}
	}
	return result, nil
}

// ForceResetIfBreached checks the password at login and forces a reset if breached.
// Returns a TokenSet with MustChangePassword=true if the password is compromised.
func (s *AuthService) ForceResetIfBreached(ctx context.Context, password string) bool {
	if s.breachChecker == nil {
		return false
	}
	breached, count, _ := s.breachChecker.IsPasswordBreached(ctx, password)
	if breached && count >= 1 {
		// Mark the credential as requiring a forced reset.
		// The login handler will return MustChangePassword=true.
		return true
	}
	return false
}
```

## 3. Password Reset as Defense

### 3.1 Proactive Password Reset

When a new breach corpus is published (e.g., Facebook 533M leak, LinkedIn 700M leak), the
IAM system should:

1. Ingest the breach corpus
2. Match against existing user credentials
3. Force password reset for affected users
4. Notify affected users

### 3.2 Breach Corpus Ingestion Pipeline

```go
package service

import (
	"bufio"
	"context"
	"crypto/sha1"
	"encoding/hex"
	"fmt"
	"os"
	"strings"
	"time"

	"github.com/redis/go-redis/v9"
)

// BreachIngestionService processes new breach corpora and triggers resets.
type BreachIngestionService struct {
	rdb           *redis.Client
	credentialRepo CredentialRepo
	notifyService *NotificationService
}

// IngestBreachFile reads a breach file (one SHA-1 hash per line) and marks
// affected users for forced password reset.
// Format: each line is an uppercase SHA-1 hash of a breached password.
func (bis *BreachIngestionService) IngestBreachFile(ctx context.Context, path, breachName string) (int, error) {
	file, err := os.Open(path)
	if err != nil {
		return 0, fmt.Errorf("open breach file: %w", err)
	}
	defer file.Close()

	affected := 0
	scanner := bufio.NewScanner(file)
	// breach hashes can be long; increase buffer.
	scanner.Buffer(make([]byte, 0, 1024*1024), 1024*1024)

	// Store breach hashes in a Redis set for fast lookup.
	breachSetKey := fmt.Sprintf("ggid:breach:%s", breachName)

	for scanner.Scan() {
		hash := strings.ToUpper(strings.TrimSpace(scanner.Text()))
		if len(hash) != 40 {
			continue
		}
		// Add to the breach set (sharded by first 2 chars for memory).
		shardKey := fmt.Sprintf("%s:%s", breachSetKey, hash[:2])
		bis.rdb.SAdd(ctx, shardKey, hash)
	}

	// Now scan all user credentials and check for matches.
	// In production, this is done in batches to avoid locking the DB.
	users, err := bis.credentialRepo.ListAllForBreachCheck(ctx)
	if err != nil {
		return 0, err
	}

	for _, cred := range users {
		// Recompute SHA-1 of the password hash prefix (we store bcrypt, so we
		// can't reverse — instead, we compare against the email:password pairs
		// from the breach by checking if the user's email is in the breach).
		_ = cred
		// In practice: check if user's email appears in the breach corpus,
		// then force a reset.
	}

	return affected, scanner.Err()
}

// ForcePasswordReset marks a user as requiring a password reset on next login
// and notifies them.
func (bis *BreachIngestionService) ForcePasswordReset(ctx context.Context, tenantID, userID string, breachName string) error {
	// Set a Redis flag that the login handler checks.
	key := fmt.Sprintf("ggid:force_reset:%s:%s", tenantID, userID)
	val := fmt.Sprintf("breach:%s:%s", breachName, time.Now().Format(time.RFC3339))
	if err := bis.rdb.Set(ctx, key, val, 0).Err(); err != nil {
		return fmt.Errorf("set force_reset flag: %w", err)
	}

	// Notify the user.
	return bis.notifyService.SendBreachNotification(ctx, tenantID, userID, breachName)
}

// IsForceResetRequired checks if a user must reset their password.
func (bis *BreachIngestionService) IsForceResetRequired(ctx context.Context, tenantID, userID string) (bool, string) {
	key := fmt.Sprintf("ggid:force_reset:%s:%s", tenantID, userID)
	val, err := bis.rdb.Get(ctx, key).Result()
	if err != nil {
		return false, ""
	}
	return true, val
}

// ClearForceReset removes the flag after the user resets their password.
func (bis *BreachIngestionService) ClearForceReset(ctx context.Context, tenantID, userID string) {
	key := fmt.Sprintf("ggid:force_reset:%s:%s", tenantID, userID)
	bis.rdb.Del(ctx, key)
}
```

### 3.3 Notification to Affected Users

```go
// NotificationService sends security alerts to users.
type NotificationService struct {
	emailSvc *EmailService
	rdb      *redis.Client
}

// SendBreachNotification emails a user about a breach affecting their account.
func (ns *NotificationService) SendBreachNotification(ctx context.Context, tenantID, userID, breachName string) error {
	// Rate-limit notifications: don't spam (max 1 per user per breach).
	dedupeKey := fmt.Sprintf("ggid:breach_notified:%s:%s:%s", tenantID, userID, breachName)
	set, err := ns.rdb.SetNX(ctx, dedupeKey, "1", 7*24*time.Hour).Result()
	if err != nil || !set {
		return nil // already notified or error — skip
	}

	// Send the email.
	subject := fmt.Sprintf("Security Alert: Your password may be compromised (%s)", breachName)
	body := fmt.Sprintf(`
A data breach ("%s") was detected that may contain your account credentials.

As a precaution, we have required a password reset for your account. Please log in
and change your password immediately.

If you use this password on other services, we strongly recommend changing it there too.

— GGID Security Team
`, breachName)

	return ns.emailSvc.Send(ctx, userID, subject, body)
}
```

## 4. Multi-Dimensional Rate Limiting for Stuffing

### 4.1 The Three Dimensions

Credential stuffing attacks manifest differently depending on the attacker's strategy.
A single rate-limiting dimension cannot catch all patterns:

| Dimension | Catches | Misses |
|---|---|---|
| Per-IP | Single-source attacks, datacenter IPs | Distributed residential proxy attacks |
| Per-account | Focused attack on one account | Broad spray across many accounts |
| Per-tenant | Tenant-wide coordinated attack | Low-volume slow attacks |

### 4.2 Distributed Rate Limiting with Pattern Detection

```go
package service

import (
	"context"
	"fmt"
	"time"

	"github.com/redis/go-redis/v9"
)

// StuffingRateLimiter combines multi-dimensional rate limits with cross-IP
// pattern detection to identify credential stuffing campaigns.
type StuffingRateLimiter struct {
	rdb *redis.Client
}

// StuffingConfig defines thresholds for stuffing detection.
type StuffingConfig struct {
	// Per-IP limits.
	IPMaxPerMinute    int // e.g., 5
	IPMaxPerHour      int // e.g., 50

	// Per-account limits.
	AccountMaxPerHour int // e.g., 10
	AccountMaxPerDay  int // e.g., 50

	// Per-tenant limits.
	TenantMaxFailedPerMinute int // e.g., 100

	// Cross-IP pattern detection.
	DistinctIPThreshold      int           // e.g., 20 distinct IPs targeting one account in 1 hour
	GlobalFailedBurstThreshold int         // e.g., 500 failed logins tenant-wide in 5 minutes
	GlobalBurstWindow         time.Duration // e.g., 5 * time.Minute

	// Low-and-slow detection.
	SlowWindow    time.Duration // e.g., 24 * time.Hour
	SlowThreshold int           // e.g., 100 failed from one IP in 24h
}

// DefaultStuffingConfig returns production-ready thresholds.
func DefaultStuffingConfig() StuffingConfig {
	return StuffingConfig{
		IPMaxPerMinute:           5,
		IPMaxPerHour:             50,
		AccountMaxPerHour:        10,
		AccountMaxPerDay:         50,
		TenantMaxFailedPerMinute: 100,
		DistinctIPThreshold:      20,
		GlobalFailedBurstThreshold: 500,
		GlobalBurstWindow:         5 * time.Minute,
		SlowWindow:                24 * time.Hour,
		SlowThreshold:             100,
	}
}

// StuffingCheckResult describes the outcome of a stuffing check.
type StuffingCheckResult struct {
	Allowed       bool
	Reason        string
	RequiresCAPTCHA bool
	BlockedIP     bool
}

// Check evaluates all dimensions and returns whether the attempt is allowed.
func (srl *StuffingRateLimiter) Check(ctx context.Context, cfg StuffingConfig, ip, account, tenantID string) StuffingCheckResult {
	now := time.Now().UnixNano()
	pipe := srl.rdb.Pipeline()

	// --- Dimension 1: Per-IP ---
	ipMinKey := fmt.Sprintf("ggid:stuffing:ip_min:%s", ip)
	ipHourKey := fmt.Sprintf("ggid:stuffing:ip_hour:%s", ip)
	pipe.Incr(ctx, ipMinKey)
	pipe.Expire(ctx, ipMinKey, time.Minute)
	pipe.Incr(ctx, ipHourKey)
	pipe.Expire(ctx, ipHourKey, time.Hour)

	// --- Dimension 2: Per-account ---
	acctHourKey := fmt.Sprintf("ggid:stuffing:acct_hour:%s:%s", tenantID, account)
	acctDayKey := fmt.Sprintf("ggid:stuffing:acct_day:%s:%s", tenantID, account)
	pipe.Incr(ctx, acctHourKey)
	pipe.Expire(ctx, acctHourKey, time.Hour)
	pipe.Incr(ctx, acctDayKey)
	pipe.Expire(ctx, acctDayKey, 24*time.Hour)

	// --- Dimension 3: Per-tenant failed attempts ---
	tenantKey := fmt.Sprintf("ggid:stuffing:tenant:%s", tenantID)
	pipe.Incr(ctx, tenantKey)
	pipe.Expire(ctx, tenantKey, time.Minute)

	// --- Dimension 4: Distinct IPs per account (cross-IP detection) ---
	acctIPSetKey := fmt.Sprintf("ggid:stuffing:acct_ips:%s:%s", tenantID, account)
	pipe.SAdd(ctx, acctIPSetKey, ip)
	pipe.Expire(ctx, acctIPSetKey, time.Hour)
	pipe.SCard(ctx, acctIPSetKey)

	// --- Dimension 5: Global burst detection ---
	globalKey := "ggid:stuffing:global_burst"
	pipe.ZAdd(ctx, globalKey, redis.Z{Score: float64(now), Member: fmt.Sprintf("%d", now)})
	pipe.ZRemRangeByScore(ctx, globalKey, "0", fmt.Sprintf("%d", now-int64(cfg.GlobalBurstWindow)))
	pipe.ZCard(ctx, globalKey)

	// --- Dimension 6: Low-and-slow per-IP ---
	slowKey := fmt.Sprintf("ggid:stuffing:slow:%s", ip)
	pipe.ZAdd(ctx, slowKey, redis.Z{Score: float64(now), Member: fmt.Sprintf("%d", now)})
	pipe.ZRemRangeByScore(ctx, slowKey, "0", fmt.Sprintf("%d", now-int64(cfg.SlowWindow)))
	pipe.ZCard(ctx, slowKey)

	cmdrs, _ := pipe.Exec(ctx)

	ipMinCount := srl.getInt(cmdrs, 0)
	ipHourCount := srl.getInt(cmdrs, 2)
	acctHourCount := srl.getInt(cmdrs, 4)
	acctDayCount := srl.getInt(cmdrs, 6)
	tenantCount := srl.getInt(cmdrs, 8)
	distinctIPs := srl.getInt(cmdrs, 11)
	globalBurst := srl.getInt(cmdrs, 13)
	slowCount := srl.getInt(cmdrs, 16)

	result := StuffingCheckResult{Allowed: true}

	// Hard blocks.
	if ipMinCount > cfg.IPMaxPerMinute {
		return StuffingCheckResult{Allowed: false, Reason: "IP rate limit exceeded", BlockedIP: true}
	}
	if acctHourCount > cfg.AccountMaxPerHour {
		return StuffingCheckResult{Allowed: false, Reason: "Account rate limit exceeded"}
	}
	if acctDayCount > cfg.AccountMaxPerDay {
		return StuffingCheckResult{Allowed: false, Reason: "Account daily limit exceeded"}
	}

	// Soft blocks — require CAPTCHA instead of hard block.
	if ipHourCount > cfg.IPMaxPerHour {
		result.RequiresCAPTCHA = true
	}
	if distinctIPs >= cfg.DistinctIPThreshold {
		// Many IPs targeting one account — classic distributed stuffing.
		result.RequiresCAPTCHA = true
	}
	if tenantCount > cfg.TenantMaxFailedPerMinute {
		result.RequiresCAPTCHA = true
	}
	if globalBurst > int64(cfg.GlobalFailedBurstThreshold) {
		// Global burst — enable CAPTCHA globally.
		result.RequiresCAPTCHA = true
	}
	if slowCount > int64(cfg.SlowThreshold) {
		// Low-and-slow attack from this IP.
		result.RequiresCAPTCHA = true
	}

	return result
}

func (srl *StuffingRateLimiter) getInt(cmdrs []redis.Cmder, idx int) int64 {
	if idx >= len(cmdrs) {
		return 0
	}
	if intCmd, ok := cmdrs[idx].(*redis.IntCmd); ok {
		return intCmd.Val()
	}
	return 0
}
```

## 5. Device Fingerprinting

### 5.1 Fingerprinting Approaches

GGID currently fingerprints devices as `SHA-256(userAgent + ip)`
(`services/auth/internal/service/device_tracking.go`). This is weak for bot detection
because:

1. User-Agent is trivially spoofed
2. IP changes on every request with proxy rotation
3. No JavaScript-based fingerprinting (canvas, WebGL, fonts)
4. No TLS fingerprinting

### 5.2 TLS Fingerprinting (JA3/JA4)

JA3 creates a fingerprint from the TLS ClientHello message: SSL version, cipher suites,
extensions, and elliptic curves. Different HTTP clients produce different JA3 hashes:

- Real Chrome: `cd08e31494f9531f560d64c695473da9`
- Python requests: `e7d705a3286e19ea42f587b344ee6865`
- Go http.Client: `19e29534fd49dd27d09234e638c98245`

JA4 is the newer version with more granularity (separate fields for hash-able components).

**Go implementation** requires intercepting the ClientHello at the TLS layer:

```go
package fingerprint

import (
	"crypto/md5"
	"crypto/sha256"
	"encoding/hex"
	"fmt"
	"net/http"
	"strings"

	"github.com/redis/go-redis/v9"
)

// FingerprintCollector collects and evaluates client fingerprints.
type FingerprintCollector struct {
	rdb            *redis.Client
	knownBotHashes map[string]string // JA3 hash → bot name
}

// NewFingerprintCollector creates a collector with known bot signatures.
func NewFingerprintCollector(rdb *redis.Client) *FingerprintCollector {
	return &FingerprintCollector{
		rdb: rdb,
		knownBotHashes: map[string]string{
			// Common bot/tool TLS fingerprints (JA3).
			"e7d705a3286e19ea42f587b344ee6865": "python-requests",
			"773906b0efdefa24a7f2b8eb6985bf37": "python-httpx",
			"5d77b935ffd1f7885dac493ca4a0ae3f": "go-http-client",
			"51c57e9c4818217f4d0c0aa5f9a5e56f": "curl",
			"b30892b3c12b6b1c0c5f11d12e2b11f9": "selenium-chrome-headless",
		},
	}
}

// ClientFingerprint aggregates all available fingerprint signals.
type ClientFingerprint struct {
	JA3Hash       string // TLS ClientHello hash
	UserAgentHash string // SHA-256 of User-Agent
	AcceptHeaders string // Accept, Accept-Language, Accept-Encoding combined
	HTTPVersion   string // HTTP/1.1, HTTP/2, HTTP/3
	HeaderOrder   string // Canonical header ordering
	Score         int    // Bot likelihood score (0-100)
	Reasons       []string
}

// FromRequest extracts fingerprint signals from an HTTP request.
func (fc *FingerprintCollector) FromRequest(r *http.Request) ClientFingerprint {
	fp := ClientFingerprint{
		HTTPVersion: r.Proto,
	}

	// User-Agent hash.
	if ua := r.UserAgent(); ua != "" {
		h := sha256.Sum256([]byte(ua))
		fp.UserAgentHash = hex.EncodeToString(h[:8])
	}

	// Accept headers — real browsers send specific patterns.
	accept := r.Header.Get("Accept")
	acceptLang := r.Header.Get("Accept-Language")
	acceptEnc := r.Header.Get("Accept-Encoding")
	fp.AcceptHeaders = fmt.Sprintf("%s|%s|%s", accept, acceptLang, acceptEnc)

	// Header ordering — bots often have different header orders.
	var headerNames []string
	for k := range r.Header {
		headerNames = append(headerNames, strings.ToLower(k))
	}
	fp.HeaderOrder = strings.Join(headerNames, ",")

	// JA3 hash would be set by a TLS interceptor at the connection level.
	// In practice, this requires a custom tls.Listener wrapper.
	// For now, we score based on available signals.

	fc.score(&fp, r)
	return fp
}

// score evaluates fingerprint signals for bot likelihood.
func (fc *FingerprintCollector) score(fp *ClientFingerprint, r *http.Request) {
	// 1. Check against known bot JA3 hashes.
	if fp.JA3Hash != "" {
		if botName, ok := fc.knownBotHashes[fp.JA3Hash]; ok {
			fp.Score = 100
			fp.Reasons = append(fp.Reasons, fmt.Sprintf("TLS fingerprint matches known bot: %s", botName))
			return
		}
	}

	// 2. Missing Accept-Language — most real browsers send this.
	if r.Header.Get("Accept-Language") == "" {
		fp.Score += 30
		fp.Reasons = append(fp.Reasons, "missing Accept-Language header")
	}

	// 3. Missing or generic Accept header.
	accept := r.Header.Get("Accept")
	if accept == "" || accept == "*/*" {
		fp.Score += 20
		fp.Reasons = append(fp.Reasons, "generic Accept header")
	}

	// 4. Headless browser indicators in User-Agent.
	ua := r.UserAgent()
	if strings.Contains(strings.ToLower(ua), "headless") {
		fp.Score += 50
		fp.Reasons = append(fp.Reasons, "headless browser detected in User-Agent")
	}
	if strings.Contains(ua, "PhantomJS") || strings.Contains(ua, "Puppeteer") {
		fp.Score += 50
		fp.Reasons = append(fp.Reasons, "automation tool in User-Agent")
	}

	// 5. Missing Sec-Fetch-* headers (Chrome 76+ sends these).
	if r.Header.Get("Sec-Fetch-Mode") == "" {
		fp.Score += 15
		fp.Reasons = append(fp.Reasons, "missing Sec-Fetch headers")
	}

	// 6. Missing cookies on a non-first-visit endpoint.
	// Real browsers accumulate cookies; bots often don't.
	if len(r.Cookies()) == 0 && r.URL.Path == "/api/v1/auth/login" {
		fp.Score += 10
		fp.Reasons = append(fp.Reasons, "no cookies on login (possible bot)")
	}

	// Cap at 100.
	if fp.Score > 100 {
		fp.Score = 100
	}
}

// IsBot returns true if the fingerprint score exceeds the threshold.
func (fp *ClientFingerprint) IsBot(threshold int) bool {
	return fp.Score >= threshold
}

// CacheFingerprint stores a fingerprint for device tracking.
func (fc *FingerprintCollector) CacheFingerprint(ctx context.Context, userID, fp string) error {
	key := fmt.Sprintf("ggid:fingerprint:%s", userID)
	return fc.rdb.SAdd(ctx, key, fp).Err()
}
```

## 6. CAPTCHA Escalation

### 6.1 Progressive CAPTCHA Strategy

Not every login needs a CAPTCHA — that hurts legitimate users. Instead, escalate based
on risk:

| Failed Attempts (per account) | Action |
|---|---|
| 0-2 | No CAPTCHA |
| 3-5 | reCAPTCHA v3 (invisible, score-based) |
| 6+ | reCAPTCHA v2 (visible challenge) or hCaptcha |
| IP flagged as bot | Always require CAPTCHA |

### 6.2 Go Code: CAPTCHA Escalation Policy

```go
package service

import (
	"context"
	"fmt"

	"github.com/redis/go-redis/v9"
)

// CAPTCHAPolicy determines when CAPTCHA is required.
type CAPTCHAPolicy struct {
	rdb *redis.Client
}

// CAPTCHATier represents the escalation level.
type CAPTCHATier int

const (
	CAPTCHANone     CAPTCHATier = 0 // no CAPTCHA needed
	CAPTCHAInvisible CAPTCHATier = 1 // reCAPTCHA v3 (score-based, invisible)
	CAPTCHAChallenge CAPTCHATier = 2 // reCAPTCHA v2 / hCaptcha (visible)
)

// EvaluateCAPTCHA determines the required CAPTCHA tier for a login attempt.
func (cp *CAPTCHAPolicy) EvaluateCAPTCHA(ctx context.Context, ip, account, tenantID string) CAPTCHATier {
	// 1. Check per-account failed attempt count.
	acctFailKey := fmt.Sprintf("ggid:captcha:acct_fail:%s:%s", tenantID, account)
	failCount, _ := cp.rdb.Get(ctx, acctFailKey).Int()

	// 2. Check per-IP failed attempt count.
	ipFailKey := fmt.Sprintf("ggid:captcha:ip_fail:%s", ip)
	ipFailCount, _ := cp.rdb.Get(ctx, ipFailKey).Int()

	// 3. Check if IP is globally flagged.
	flaggedKey := fmt.Sprintf("ggid:captcha:flagged_ip:%s", ip)
	flagged := cp.rdb.Exists(ctx, flaggedKey).Val() > 0

	// Decision logic.
	if flagged {
		return CAPTCHAChallenge
	}

	totalFail := failCount + ipFailCount
	switch {
	case totalFail >= 6:
		return CAPTCHAChallenge
	case totalFail >= 3:
		return CAPTCHAInvisible
	default:
		return CAPTCHANone
	}
}

// RecordFailedAttempt increments counters used by the CAPTCHA policy.
func (cp *CAPTCHAPolicy) RecordFailedAttempt(ctx context.Context, ip, account, tenantID string) {
	acctKey := fmt.Sprintf("ggid:captcha:acct_fail:%s:%s", tenantID, account)
	ipKey := fmt.Sprintf("ggid:captcha:ip_fail:%s", ip)

	pipe := cp.rdb.Pipeline()
	pipe.Incr(ctx, acctKey)
	pipe.Expire(ctx, acctKey, 3600) // 1 hour window
	pipe.Incr(ctx, ipKey)
	pipe.Expire(ctx, ipKey, 3600)
	pipe.Exec(ctx)
}

// RecordSuccessfulLogin resets counters.
func (cp *CAPTCHAPolicy) RecordSuccessfulLogin(ctx context.Context, ip, account, tenantID string) {
	acctKey := fmt.Sprintf("ggid:captcha:acct_fail:%s:%s", tenantID, account)
	ipKey := fmt.Sprintf("ggid:captcha:ip_fail:%s", ip)
	cp.rdb.Del(ctx, acctKey, ipKey)
}

// VerifyCAPTCHAToken verifies a reCAPTCHA v3 token with Google's API.
// Returns the score (0.0-1.0) and whether it passes the threshold.
func VerifyCAPTCHAToken(ctx context.Context, secret, token, remoteIP string, minScore float64) (float64, bool, error) {
	// POST to https://www.google.com/recaptcha/api/siteverify
	// with secret, response (token), and remoteip.
	// Parse the JSON response for "score" and "success".
	//
	// Example response:
	// {
	//   "success": true,
	//   "score": 0.87,
	//   "action": "login",
	//   "challenge_ts": "2024-01-01T12:00:00Z",
	//   "hostname": "example.com"
	// }
	//
	// For production, use http.Client with timeout and verify:
	//   - success == true
	//   - score >= minScore (typically 0.5)
	//   - action == expected action ("login")
	//   - hostname matches expected domain
	return 0.87, true, nil // placeholder
}

// FlagIP marks an IP as requiring always-CAPTCHA (e.g., detected bot).
func (cp *CAPTCHAPolicy) FlagIP(ctx context.Context, ip string) {
	key := fmt.Sprintf("ggid:captcha:flagged_ip:%s", ip)
	cp.rdb.Set(ctx, key, "1", 24*3600) // 24 hours
}
```

### 6.3 CAPTCHA Bypass Detection

Sophisticated attackers use CAPTCHA-solving services (2Captcha, Anti-Captcha) at ~$2 per
1000 solves. Detection strategies:

1. **Solve time analysis**: Real humans take 3-15 seconds. Instant solves (<1s) indicate
   automated solving with pre-fetched tokens.
2. **Token reuse**: Track token uniqueness; reject replayed tokens.
3. **Score distribution**: reCAPTCHA v3 scores cluster at 0.9 for real users. If many
   logins from one IP have scores of exactly 0.5-0.6 (the minimum), it suggests solving
   service tokens.
4. **Volume vs solves**: If an IP solves 100 CAPTCHAs but only 5 succeed at login, it's
   likely automated.

## 7. Anomaly Detection for Login

### 7.1 GGID's Existing Anomaly Detection

### 7.2 Additional Signals for Stuffing Detection

```go
package service

import (
	"context"
	"fmt"
	"time"

	"github.com/redis/go-redis/v9"
)

// LoginAnomalyScorer extends GGID's anomaly detection with stuffing-specific signals.
type LoginAnomalyScorer struct {
	rdb *redis.Client
}

// AnomalyScore holds the combined anomaly assessment.
type AnomalyScore struct {
	Score         int
	GeoVelocity   bool   // impossible travel
	NewASN        bool   // login from new autonomous system
	UnusualHour   bool   // anomalous time-of-day
	DistributedAttack bool // many IPs targeting this account
	Reasons       []string
}

// Score evaluates login anomalies specific to credential stuffing.
func (las *LoginAnomalyScorer) Score(ctx context.Context, userID, ip string, lat, lon float64) AnomalyScore {
	result := AnomalyScore{}

	// 1. Geographic velocity (impossible travel).
	// If the user logged in from location A 1 hour ago and now from location B
	// that is 10,000km away, it's impossible (unless they're on a plane).
	lastLoginKey := fmt.Sprintf("ggid:anomaly:lastgeo:%s", userID)
	lastGeo, _ := las.rdb.HGetAll(ctx, lastLoginKey).Result()
	if len(lastGeo) > 0 {
		var lastLat, lastLon float64
		var lastTimeStr string
		fmt.Sscanf(lastGeo["coords"], "%f,%f", &lastLat, &lastLon)
		lastTimeStr = lastGeo["time"]

		lastTime, _ := time.Parse(time.RFC3339, lastTimeStr)
		elapsed := time.Since(lastTime)

		dist := haversineDistance(lat, lon, lastLat, lastLon)
		// Max realistic speed: 1000 km/h (commercial jet).
		maxDist := elapsed.Hours() * 1000
		if dist > maxDist && dist > 500 {
			result.GeoVelocity = true
			result.Score += 50
			result.Reasons = append(result.Reasons,
				fmt.Sprintf("impossible travel: %.0f km in %v", dist, elapsed))
		}
	}

	// 2. New ASN (autonomous system number).
	// Different ASN = different network provider. A sudden change from ASN 12345
	// (Comcast) to ASN 67890 (AWS) is suspicious.
	// In production, use a GeoIP database (MaxMind) to resolve ASN from IP.
	asnKey := fmt.Sprintf("ggid:anomaly:asn:%s", userID)
	knownASN, _ := las.rdb.Get(ctx, asnKey).Result()
	currentASN := resolveASN(ip) // stub: use MaxMind GeoIP2 in production
	if knownASN != "" && currentASN != "" && knownASN != currentASN {
		result.NewASN = true
		result.Score += 20
		result.Reasons = append(result.Reasons,
			fmt.Sprintf("login from new ASN %s (was %s)", currentASN, knownASN))
	}

	// 3. Unusual time-of-day (based on user's timezone).
	// A user who always logs in 9 AM - 6 PM local suddenly logs in at 3 AM.
	userTzKey := fmt.Sprintf("ggid:anomaly:tz:%s", userID)
	userTz, _ := las.rdb.Get(ctx, userTzKey).Result()
	if userTz != "" {
		loc, err := time.LoadLocation(userTz)
		if err == nil {
			hour := time.Now().In(loc).Hour()
			if hour < 4 || hour > 23 {
				result.UnusualHour = true
				result.Score += 15
				result.Reasons = append(result.Reasons,
					fmt.Sprintf("login at unusual hour %d:00 (%s)", hour, userTz))
			}
		}
	}

	// 4. Distributed attack detection — many distinct IPs targeting this account.
	ipSetKey := fmt.Sprintf("ggid:anomaly:acct_ips:%s", userID)
	las.rdb.SAdd(ctx, ipSetKey, ip)
	las.rdb.Expire(ctx, ipSetKey, time.Hour)
	distinctIPs, _ := las.rdb.SCard(ctx, ipSetKey).Result()
	if distinctIPs >= 10 {
		result.DistributedAttack = true
		result.Score += 40
		result.Reasons = append(result.Reasons,
			fmt.Sprintf("%d distinct IPs attempted login for this account in 1 hour", distinctIPs))
	}

	// Cap score.
	if result.Score > 100 {
		result.Score = 100
	}

	return result
}

// resolveASN is a stub. In production, use MaxMind GeoIP2 ASN database.
func resolveASN(ip string) string {
	return "AS12345"
}
```

## 8. Credential Stuffing Bot Detection

### 8.1 Bot Signatures

Credential stuffing bots exhibit recognizable patterns:

| Signal | Real Browser | Bot |
|---|---|---|
| Cookies | Accumulates cookies across requests | Often cookieless or session-stripped |
| User-Agent | Consistent, browser-specific | Same UA across thousands of IPs, or rotating |
| Accept-Language | Present, matches locale | Missing, or always "en-US" regardless of geo |
| Sec-Fetch-* | Present (Chrome 76+) | Missing |
| HTTP/2 | Uses HTTP/2 with browser-specific SETTINGS | May use HTTP/1.1 or HTTP/2 with library defaults |
| TLS JA3/JA4 | Browser-specific fingerprint | Library-specific fingerprint (Go, Python, curl) |
| Timing | Variable, human-paced (3-30s between requests) | Fixed intervals, often <1s |
| Favicon/JS/CSS | Requests static assets | Only hits API endpoints |

### 8.2 Go Code: Bot Detection Middleware

```go
package middleware

import (
	"context"
	"fmt"
	"net/http"
	"strings"
	"time"

	"github.com/redis/go-redis/v9"
)

// BotDetector is HTTP middleware that scores requests for bot likelihood.
type BotDetector struct {
	rdb             *redis.Client
	scoreThreshold  int
	blockThreshold  int
}

// NewBotDetector creates middleware. scoreThreshold triggers CAPTCHA;
// blockThreshold triggers a hard 403.
func NewBotDetector(rdb *redis.Client, scoreThreshold, blockThreshold int) *BotDetector {
	return &BotDetector{
		rdb:            rdb,
		scoreThreshold: scoreThreshold,
		blockThreshold: blockThreshold,
	}
}

// Middleware wraps the next handler with bot detection.
func (bd *BotDetector) Middleware(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		// Only check auth endpoints.
		if !strings.HasPrefix(r.URL.Path, "/api/v1/auth/") {
			next.ServeHTTP(w, r)
			return
		}

		score := bd.scoreRequest(r)

		// Store score for analytics.
		ip := extractIP(r)
		scoreKey := fmt.Sprintf("ggid:bot_score:%s", ip)
		bd.rdb.Set(context.Background(), scoreKey, score, time.Hour)

		if score >= bd.blockThreshold {
			w.Header().Set("Content-Type", "application/json")
			w.WriteHeader(http.StatusForbidden)
			fmt.Fprint(w, `{"error":"access_denied","reason":"automated_traffic_detected"}`)
			return
		}

		if score >= bd.scoreThreshold {
			// Signal downstream to require CAPTCHA.
			w.Header().Set("X-Captcha-Required", "true")
		}

		next.ServeHTTP(w, r)
	})
}

// scoreRequest evaluates HTTP request signals for bot likelihood.
func (bd *BotDetector) scoreRequest(r *http.Request) int {
	score := 0

	// 1. Missing or suspicious User-Agent.
	ua := r.UserAgent()
	if ua == "" {
		score += 25
	} else {
		// Check for known automation tools.
		lowerUA := strings.ToLower(ua)
		botIndicators := []string{
			"headless", "phantomjs", "puppeteer", "selenium",
			"webdriver", "nightmare", "playwright",
		}
		for _, indicator := range botIndicators {
			if strings.Contains(lowerUA, indicator) {
				score += 60
				break
			}
		}
		// Check for library UAs (python-requests, Go-http-client, curl, etc.).
		libIndicators := []string{
			"python-requests", "go-http-client", "curl/",
			"okhttp", "java/", "apache-httpclient", "node-fetch",
		}
		for _, indicator := range libIndicators {
			if strings.Contains(lowerUA, indicator) {
				score += 50
				break
			}
		}
	}

	// 2. Missing standard browser headers.
	requiredHeaders := []string{
		"Accept", "Accept-Language", "Accept-Encoding",
	}
	missingCount := 0
	for _, h := range requiredHeaders {
		if r.Header.Get(h) == "" {
			missingCount++
		}
	}
	score += missingCount * 15

	// 3. Missing Sec-Fetch headers (modern browsers always send these).
	if r.Header.Get("Sec-Fetch-Mode") == "" {
		score += 15
	}
	if r.Header.Get("Sec-Ch-Ua") == "" {
		// Chrome 89+ sends Sec-CH-UA client hints.
		if strings.Contains(ua, "Chrome") && !strings.Contains(ua, "Edg") {
			score += 10
		}
	}

	// 4. No cookies on second+ request (real browsers accumulate cookies).
	if len(r.Cookies()) == 0 {
		// First visit is OK; but if this IP has made many requests with no
		// cookies, it's suspicious.
		ip := extractIP(r)
		nocookieKey := fmt.Sprintf("ggid:nocookie:%s", ip)
		count, _ := bd.rdb.Incr(context.Background(), nocookieKey).Result()
		if count == 1 {
			bd.rdb.Expire(context.Background(), nocookieKey, time.Hour)
		}
		if count > 5 {
			score += 20
		}
	}

	// 5. Timing analysis — check request interval from this IP.
	ip := extractIP(r)
	lastReqKey := fmt.Sprintf("ggid:lastreq:%s", ip)
	lastReqStr, _ := bd.rdb.Get(context.Background(), lastReqKey).Result()
	bd.rdb.Set(context.Background(), lastReqKey, time.Now().UnixNano(), time.Minute)
	if lastReqStr != "" {
		var lastReqNs int64
		fmt.Sscanf(lastReqStr, "%d", &lastReqNs)
		interval := time.Since(time.Unix(0, lastReqNs))
		// Very fast sequential requests (<500ms) suggest automation.
		if interval < 500*time.Millisecond {
			score += 20
		}
	}

	// 6. Identical User-Agent across many IPs (check global UA frequency).
	if ua != "" {
		uaHashKey := fmt.Sprintf("ggid:ua_freq:%s", hashString(ua))
		freq, _ := bd.rdb.Incr(context.Background(), uaHashKey).Result()
		if freq == 1 {
			bd.rdb.Expire(context.Background(), uaHashKey, time.Hour)
		}
		// If a single UA string appears from 50+ distinct IPs in an hour,
		// it's likely a botnet using the same UA.
		ipCountKey := fmt.Sprintf("ggid:ua_ips:%s", hashString(ua))
		bd.rdb.SAdd(context.Background(), ipCountKey, ip)
		bd.rdb.Expire(context.Background(), ipCountKey, time.Hour)
		distinctIPs, _ := bd.rdb.SCard(context.Background(), ipCountKey).Result()
		if distinctIPs > 50 {
			score += 30
		}
	}

	if score > 100 {
		score = 100
	}
	return score
}

func extractIP(r *http.Request) string {
	if xff := r.Header.Get("X-Forwarded-For"); xff != "" {
		parts := strings.SplitN(xff, ",", 2)
		return strings.TrimSpace(parts[0])
	}
	host := r.RemoteAddr
	if idx := strings.LastIndex(host, ":"); idx > 0 {
		host = host[:idx]
	}
	return host
}

func hashString(s string) string {
	h := time.Now().Unix()
	_ = h
	return fmt.Sprintf("%x", len(s))
}
```

```go
Your password was found in the "%s" data breach (appeared %d times).

Your account security may be at risk. Please change your password immediately at:
  https://your-app.com/settings/password

Tips for a strong password:
  - Use a unique password for each account
  - Consider using a password manager
  - Enable two-factor authentication

— GGID Security Team
`, breachName, count)

	return ln.emailSvc.Send(ctx, email, subject, body)
}

// EducationReminder sends periodic password-reuse education to users who haven't
// changed their password in over a year.
func (ln *LoginNotifier) EducationReminder(ctx context.Context, email, lastChangedStr string) error {

	body := fmt.Sprintf(`

	return ln.emailSvc.Send(ctx, email, subject, body)
}
```

## 10. GGID Credential Stuffing Defense Audit

### 10.1 What GGID Currently Has

Based on review of the GGID source code (`services/auth/` and `services/gateway/`):

| Defense | Status | Location |
|---|---|---|
| **Per-IP rate limiting** | EXISTS | `auth_service.go` Login(): `login:{ip}` → 5/min |
| **Per-IP brute-force (sliding window)** | EXISTS | `auth_service.go` CheckBruteForce(): 20/min per IP |
| **Per-account rate limiting** | EXISTS | `auth_service.go` CheckBruteForce(): 10/hour per user |
| **Account lockout** | EXISTS | `auth_service.go` IsAccountLocked/RecordFailedLogin |
| **Anomaly lockout** | EXISTS | `anomaly_detection.go` RecordFailedLoginAnomaly: 5 failures → 15min lock |
| **Geo anomaly detection** | EXISTS | `anomaly_detection.go` CheckGeoAnomaly: haversine >500km |
| **New device detection** | EXISTS | `anomaly_detection.go` CheckNewDevice |
| **Risk-based scoring** | EXISTS | `risk_auth.go` AssessLoginRisk: 0-100 score, step-up auth |
| **IP blocklist** | EXISTS | `risk_auth.go` BlockSuspiciousIP/IsIPBlocked |
| **Multi-user brute-force detection** | EXISTS | `risk_auth.go` RecordFailedLoginAttempt: ≥3 users from one IP |
| **Login attempt logging** | EXISTS | `login_attempt.go` RecordLoginAttempt → Redis sorted set |
| **Device fingerprint (UA+IP hash)** | EXISTS | `device_tracking.go` SHA-256(UA+IP) |

| **Gateway token bucket** | EXISTS | `gateway/middleware/token_bucket.go` tier-based |
| **Password breach check (registration)** | EXISTS | `password_breach.go` HIBP k-anonymity API |

### 10.2 What GGID Is Missing

| Missing Defense | Priority | Impact |
|---|---|---|
| **Breach check at LOGIN** | P0 | Existing users with breached passwords are unprotected |
| **Per-tenant rate limiting at auth level** | P1 | Can't detect tenant-wide stuffing campaigns |
| **CAPTCHA escalation** | P1 | No progressive challenge for suspicious activity |
| **TLS JA3/JA4 fingerprinting** | P1 | Can't distinguish real browsers from automated tools |
| **Bot detection middleware** | P1 | No systematic bot signature checking |
| **Breach corpus ingestion pipeline** | P2 | No proactive password resets for new breaches |
| **Client-side browser integrity check** | P2 | No canvas/WebGL/webdriver detection |
| **Login notification system** | P2 | Users not alerted on new device login |
| **Low-and-slow detection (24h window)** | P2 | Existing 15-min window misses slow attacks |
| **Cross-IP distributed attack detection** | P2 | No "many IPs, one account" pattern matching |
| **Sec-Fetch/Sec-CH-UA header validation** | P3 | No modern browser signal checking |
| **Request timing analysis** | P3 | No fixed-interval bot detection |

### 10.3 Gateway vs Auth Service Gap

The **gateway** (`services/gateway/internal/middleware/ratelimit.go`) uses an **in-memory**
rate limiter with fixed windows. This means:

1. Rate limits reset on gateway restart
2. Multiple gateway instances don't share state
3. No coordination across instances

The **auth service** rate limiter (`ratelimit_service.go`) is **Redis-backed**, which is
correct for distributed deployments. The gateway should either use the same Redis-backed
limiter or forward auth requests to the auth service for rate limiting.

### 10.4 Key Observation: Breach Check Disconnect

The `CheckPasswordBreach` method is implemented on `PasswordService` but is **only called
during registration** (in the password validation chain). It is **never called during
`Login`** or `LoginMFA`. This is the single biggest gap — a user who registered before the
breach check was added, or whose password appeared in a breach after registration, can log
in indefinitely without being warned or forced to reset.

## 11. Gap Analysis and Recommendations

### 11.1 Priority Action Items

#### P0: Add Breach Check at Login (Effort: 1-2 days)

**What**: After successful authentication in `AuthService.Login()`, call
`breachChecker.IsPasswordBreached(ctx, password)`. If breached, set
`MustChangePassword: true` and return a short-lived reset token.

**Why**: This is the single highest-impact change. Credential stuffing succeeds because
users reuse breached passwords. Detecting this at login closes the loop for existing users.

**Files to modify**:
- `services/auth/internal/service/auth_service.go` — add breach check after auth
- `services/auth/internal/service/password_breach.go` — add caching layer
- `services/auth/internal/server/http.go` — handle `MustChangePassword` response

#### P1: Implement Multi-Dimensional Stuffing Rate Limiter (Effort: 3-5 days)

**What**: Replace the current single-dimension rate check in `Login()` with the
`StuffingRateLimiter` described in section 4. Add per-tenant, cross-IP, and low-and-slow
detection.

**Why**: Current rate limits catch single-source attacks but miss distributed stuffing
from residential proxies. The cross-IP detection (distinct IPs targeting one account) is
the key missing signal.

**Files to modify**:
- New file: `services/auth/internal/service/stuffing_ratelimit.go`
- `services/auth/internal/service/auth_service.go` — wire in the new limiter

#### P1: Add CAPTCHA Escalation (Effort: 2-3 days)

**What**: Implement the `CAPTCHAPolicy` from section 6. Integrate reCAPTCHA v3
(score-based, invisible) for medium-risk attempts and reCAPTCHA v2/hCaptcha for high-risk.

**Why**: Rate limiting alone can't stop distributed attacks without also blocking
legitimate users. CAPTCHA escalation adds a challenge layer that stops bots while
minimizing friction for real users.

**Files to create**:
- `services/auth/internal/service/captcha_policy.go`
- Frontend: add reCAPTCHA script and token submission
- `services/auth/internal/server/http.go` — verify token before login

#### P1: Add Bot Detection Middleware (Effort: 3-5 days)

**What**: Implement the `BotDetector` middleware from section 8 as gateway middleware.
Check for missing browser headers, automation tool indicators, timing patterns, and
TLS JA3/JA4 fingerprints.

**Why**: The gateway is the right layer for bot detection because it sees all traffic
before it reaches auth. Early rejection saves auth-service resources.

**Files to create**:
- `services/gateway/internal/middleware/bot_detection.go`
- Wire into gateway middleware chain in `cmd/main.go`

#### P2: Implement Breach Corpus Ingestion Pipeline (Effort: 5-7 days)

**What**: Build the `BreachIngestionService` from section 3. Subscribe to breach feeds
(HIBP, DeHashed), match affected users, and force password resets with notifications.

**Why**: Proactive defense is better than reactive. When a new breach is published, GGID
should automatically protect affected users within hours, not wait for them to log in.

**Files to create**:
- `services/auth/internal/service/breach_ingestion.go`
- Background job in `cmd/main.go` for periodic feed checks
- Admin API endpoint to trigger manual ingestion

### 11.2 Effort Summary

| Item | Priority | Effort | Impact |
|---|---|---|---|
| Breach check at login | P0 | 1-2 days | Critical — closes biggest gap |
| Multi-dimensional rate limiter | P1 | 3-5 days | High — catches distributed attacks |
| CAPTCHA escalation | P1 | 2-3 days | High — stops bots without blocking users |
| Bot detection middleware | P1 | 3-5 days | High — early rejection at gateway |
| Breach corpus ingestion | P2 | 5-7 days | Medium — proactive defense |
| **Total** | | **14-22 days** | |

### 11.3 Architecture Recommendation

```
Request → Gateway
             ├─ Bot Detection Middleware (score 0-100)
             │    ├─ Block if score ≥ 80 (403)
             │    └─ Flag for CAPTCHA if score ≥ 50
             ├─ Rate Limiter (Redis-backed, multi-dimensional)
             │    ├─ Per-IP, Per-account, Per-tenant
             │    └─ Cross-IP pattern detection
             └─ Proxy → Auth Service
                          ├─ Breach Check (cached HIBP)
                          ├─ Anomaly Scorer (geo, device, ASN, velocity)
                          ├─ CAPTCHA Policy (escalation tiers)
                          ├─ Authenticate
                          └─ Login Notifier (new device, geo anomaly)
```

### 11.4 Metrics to Track

Implement dashboards for:

1. **Failed login rate** per minute (baseline + spike detection)
2. **Distinct IPs per account** per hour (stuffing indicator)
3. **CAPTCHA solve rate** (high solve rate + high failure = automated solving service)
4. **Breach detection rate** at login (% of logins with breached passwords)
5. **Bot score distribution** (shift toward high scores = campaign)
6. **Geographic velocity alerts** (impossible travel events per day)
7. **Force-reset completion rate** (% of flagged users who actually reset)
